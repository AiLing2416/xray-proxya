xray-proxya v0.2.0 更新日志 - 高级代理与多租户管理
============================================================

本版本是项目从 Bash 迁移到 Go 后的第一个里程碑式大版本，引入了核心的租户隔离机制与底层运行时加固。

一、 核心特性：Guests 租户管理系统 (Heavyweight)
------------------------------------------------------------
1.  【多用户隔离】支持 guests add/del/list 命令，为每个租户生成独立 UUID。
2.  【配额管理】支持 --quota 参数设置 GB 级传输配额（仅计下行流量）。
    - 支持 -1（无限）、0（暂停使用）、reset（用量清零）。
    - 支持 --reset 指定每月固定重置日（具备月末边界自适应算法）。
3.  【专属路由 (Per-Guest Outbound)】支持 guests set --outbound 'SHARE_LINK'。
    - 允许为特定租户分配专用出口（如 Shadowsocks 或 VLESS 转发节点）。
    - 自动生成 Xray 路由规则，实现基于 User 标识的精准流控。
4.  【快捷分发】guests show 支持基于服务器现有预设自动生成租户专属分享链接。

二、 高级出站与接口绑定
------------------------------------------------------------
1.  【物理接口绑定】outbound bind-interface 命令。
    - 支持将 freedom 协议绑定至特定的本地网络接口（如 WireGuard/proton-vpn）。
    - 自动探测接口 IP 并配置 sendThrough，实现基于接口的策略出站。
2.  【内网代理中转】outbound set-internal-proxy 命令。
    - 为任意出站节点（Relay 或接口绑定）创建私有的 Socks/HTTP 本地代理端口。

三、 运行时稳定性与交互逻辑加固
------------------------------------------------------------
1.  【架构修正】彻底修复了子命令注册逻辑，确保 help 列表包含全部 20+ 个核心指令。
2.  【补全进化】
    - 转向动态生成模式，确保 guests 子命令及别名在任何设备上 100% 同步。
    - 增加 guest -> guests 等单复数容错别名补全。
3.  【进程管理自愈】
    - 修复了 apply 阶段的“自杀”Bug：改用 pkill -x xray 精确匹配，防止误杀 xray-proxya。
    - 引入 start 命令：支持跳过冗长校验直接拉起后台服务。
    - 路径回退机制：当 os.Executable() 失效时自动回退至 Args[0]，消除 exec: no command 报错。
4.  【环境零依赖】
    - 核心下载器改用纯 Go (archive/zip) 实现，彻底告别对系统 unzip 命令的依赖。
    - 支持在极简/纯净版 Linux 系统（如 Alpine/Debian Slim）上开箱即用。

四、 安全特性
------------------------------------------------------------
1.  【端口随机化】init 时 API 和测试端口在 10000-65535 范围内随机抽取，极大增强了多用户部署的兼容性。
2.  【链接防断保护】内置 Shell 截断检测，当用户忘记加引号导致链接解析不全时提供预警并拦截写入。

五、 增强型透明代理与分权监听 (Gateway Evolution)
------------------------------------------------------------
1.  【分权监听】引入 LocalEnabled 与 LANEnabled 独立开关。
    - 允许仅开启本机透明代理而不影响局域网流量，反之亦然。
2.  【防失联加固】SyncFirewall 自动探测系统 SSH 端口。
    - nftables 规则自动排除所有活动的 SSH 端口，防止透明代理配置错误导致用户失联。
3.  【TUN 本机接管】针对本机代理引入现代化的 TUN 模式方案。
    - 当 Mode 为 tun 时，利用 Xray 原生 autoRoute/strictRoute 技术接管本机流量，性能更优且无需复杂 nftables 规则。
4.  【规则闭环自愈】SyncFirewall 与 CleanupFirewall 深度集成至服务生命周期。
    - 支持开机自启自动恢复内核规则，服务停止自动清理规则，确保系统环境纯净。

v0.2.0 确立了“用户自治、逻辑隔离、运行时自愈”的核心架构准则。
